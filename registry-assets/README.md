# Registry Assets

This directory is populated by **Phase 7** of the Rocketing cascade pipeline. The coding agent (Phase 8) reads these assets when customizing the app.

## How It Works

1. Phase 7 (`populateRepo`) matches this app's requirements against the Rocketing registries
2. Matched assets are downloaded and committed into `registry-assets/{collection}/{name}/`
3. A `manifest.json` is written to this directory listing all available assets
4. The Phase 8 coding agent reads the manifest and integrates assets into the app

## Expected Structure

```
registry-assets/
├── manifest.json                    # Auto-generated by Phase 7
├── components/
│   ├── data-table/                  # Component from registry
│   │   ├── data-table.tsx
│   │   └── package.json
│   └── form-fields/
│       └── ...
├── blocks/
│   ├── hero-section/
│   └── pricing-section/
├── layouts/
│   └── dashboard-layout/
└── themes/
    └── brand-theme/
```

## Manifest Format

```json
{
  "version": "1.0.0",
  "generatedAt": "2026-02-24T00:00:00Z",
  "cascadeId": "cascade_xxx",
  "assets": [
    {
      "name": "data-table",
      "collection": "components",
      "version": "1.0.0",
      "path": "components/data-table/",
      "registryUrl": "https://registry.rocketing.ai/api/components/data-table",
      "status": "downloaded"
    }
  ]
}
```

## For Coding Agents

- Check `manifest.json` to see what's available
- Assets are pre-downloaded — no need to fetch from the registry API
- Follow each asset's `package.json` or README for integration instructions
- Update manifest status to "integrated" after installing an asset
